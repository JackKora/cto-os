# Code Contribution Opportunities

**Scope:** Identifying and tracking the user's own hands-on technical contribution. Scans open PRs / tickets / reliability gaps to surface opportunities that match declared preferences (architectural, knotty bugs, learning, reliability, platform). Logs contributions actually picked up. Shows trend vs. target cadence. Optional-by-role module — for leaders who want to stay technical.

**Out of scope:** Strategic technical direction (Technical Strategy); routine PR review as coaching (Managing Down); reliability operations (Tech Ops — though reliability gaps surface here as opportunities); team-level throughput metrics (Process Management / Team Management).

**Frameworks:** [Donald Reinertsen — *The Principles of Product Development Flow*](https://www.amazon.com/Principles-Product-Development-Flow-Generation/dp/1935401009), applied as an opportunity-spotting lens ("where is this senior-hand contribution worth most").

**Depends on:**
- Required: none
- Optional: `personal-os` (stated intent — "stay technical" shifts aggressiveness of surfacing), `technical-strategy` (tech debt items the user could personally take), `tech-ops` (reliability gaps as opportunities), `process-management`

**Example tasks:**
- "Scan the PRs open this week — where should I dive in?"
- "Find me an architecturally interesting ticket to take this sprint."
- "Log the fix I just shipped — it was a knotty one in the platform."
- "Am I on target for hands-on this quarter?"
- "Add [repo] to my contribution sources."

**State location:** `cto-os-data/modules/code-contribution/state/`
